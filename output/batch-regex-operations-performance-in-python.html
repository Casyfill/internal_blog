Batch Regex operations: performance in python

Sat 30 June 2018 by Philipp Kats

There was a discussion recently on whether it is better to use pandas native interface to regex over iterating; As I've been able to imagine even more different angles of attack (say, using regex in SQL, or merging all patterns into one), I decided to check out them all and see how performance is different

TL;DR;¶

  • slightly concize version of SE.concessions. (2 min 42 sec +/- 6 sec)
  • merged patterns in each deck and precompiled them (2min 45s ± 1.26 s)
  • Next, we checked how apply behaves (2min 44s ± 280 ms)
  • as well as pandas native str.contains (2min 39s ± 4 s)
  • we also tried to use dask to leverage our multiple cores (3min 7s ± 31.2 s) on 3 cores
  • and finally, used SQL regex implementations to run computations on the server - was too long to wait
  • In short, none of the attempts resulted in any significant improvement, likely because all of them use optimized C code under the hood, anyway. Yet, vectorized computations results in a way cleaner code.
  • Dask's overhead resulted in significant increase for this amount of data.
  • Using regex as part of SQL query leads to a very long querying time - of course, with the advantage of machine-independent results

patterns = {'rent':rent_patterns, 
            "cash": cash_patterns,

def get_rental_concession(description):
    '''given a rental listing description, determine whether listing contains
        one or more concessions and what type of concessions it contains

    Args:
        description(str): listing description
    Returns:
        ctypes: list of categories of concession contained
    '''
    tags = set()
    
    for k, pattern_deck in patterns.items():
        for pattern in pattern_deck:
            if re.search(pattern, description):
                tags.add(k)
                break
    
    if tags:
        return tags
